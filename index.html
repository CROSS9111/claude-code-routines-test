Claude Code 向け無料セキュリティガイダンスプラグインをリリース ─ SQLi・XSS・シークレットハードコードなど25クラスを3層でリアルタイム検出
New Plugin Source: CyberSecurityNews / Help Net Security / SecurityWeek / AI Weekly — 2026.05.27

Anthropic は5月27日、Claude Code の全プラン・全ユーザーに向けて無料のセキュリティガイダンスプラグインをリリースした。/plugins コマンドからプラグインマーケットプレイスに直接インストールでき、コードを書きながらリアルタイムで脆弱性パターンを検出する。

  • 検出対象（約25クラス）: SQL インジェクション、コマンドインジェクション、XSS、ハードコードされた API キーとシークレット、セキュアでないデシリアライゼーション、不適切な入力バリデーションなど
  • 3層レビュー体制: ①ファイル編集時、②モデルターン後、③コミット前、の3段階でコードをチェック。問題が検出されると、同一セッション内でインラインフィックスを提案する
  • 実績: 内部テストで PR のセキュリティ関連コメントを30〜40%削減。リリース24時間で157,000ダウンロードを記録し、Medium での独立したテスト記事（Joe Njenga 氏）でも「実際に隠れたバグを発見できた」と報告されている
  • Claude Sandbox との併用: 同日リリースの Claude Sandbox（コードをサンドボックス環境で実行してリスクを最小化する機能）と組み合わせることで、コーディングから実行まで一貫したセキュリティカバレッジが得られる
Dynamic Workflows 発表翌日の反響 ─ Bun の75万行 Rust 移植が「本当に動いた」ことで業界に衝撃、HN では制御機構への議論も
Reaction Source: Hacker News / ClaudeFa.st / TestingCatalog — 2026.05.28-29

昨日（5/28）発表された Dynamic Workflows は、Bun の創設者 Jarred Sumner による Zig→Rust 全面移植（約75万行・11日間・既存テスト99.8%パス）を旗艦事例として大きな反響を呼んでいる。

  • ワークフローの内訳: ①全 Zig 構造体フィールドに適切な Rust ライフタイムをマッピングする並列エージェント群、②各 .zig ファイルを動作等価な .rs ファイルに変換する並列エージェント群（各ファイルにレビュアー2体）、③ビルドとテストが通るまでループで修正する検証エージェント群
  • Hacker News の主要議論: 「長時間実行セッションへの割り込み手段が不足している」「途中経過のコードレビューを逐次挿入する仕組みが欲しい」という制御機構への要望が多い。上限16並列エージェント・計1,000エージェントという制約についても「十分か」という声も
  • 利用要件: Max・Team・API プランはデフォルト有効。Enterprise プランはデフォルト無効（管理者が設定で有効化可能）。Free プランは対象外
  • 「普通の開発者が75万行のリファクタリングを11日でできるとは思えなかったが、これが現実になった。次の問いは『何をエージェントに任せ、何を手で書くか』だ」という声が高評価を集めている
MIT Technology Review「Code with Claude 2026 は開発の未来を見せた ─ 好むと好まざるとにかかわらず」
Analysis Source: MIT Technology Review — 2026.05.21

MIT Technology Review は5月21日付けで、5月6日にサンフランシスコで開催された Anthropic の開発者カンファレンス「Code with Claude 2026」を取り上げ、「AIコーディングの未来を見せた ─ 好むと好まざるとにかかわらず」と評した。

  • カンファレンスの主要テーマ: Anthropic が提示した「開発者の役割の変化」。単一エージェントへの指示から、エージェントチームの設計・監督へのシフトが強調された
  • 三層構造の提示: ①ツール層（Claude Code CLI・MCP・Hooks）、②オーケストレーション層（Dynamic Workflows・Agent View）、③監督層（人間が担う設計・判断・倫理）という三層モデルが示された
  • 批判的な視点: 記事は「一部の開発者にとっては福音だが、コーディングの喜びや手作りの感覚を失うと感じる人も多い」と指摘。「ツールを使うかどうかの選択権は人間にあるが、使わない選択のコストは年々高まっている」と締めくくった
  • Anthropic は同カンファレンスで Dynamic Workflows・Agent View・セキュリティプラグインの三点セットを予告しており、5/28-29での立て続けのリリースはその実行となる

OpenAI、フロンティアガバナンスフレームワーク公開 ─ EU AI Act・カリフォルニア州透明性法への対応、リスクをサイバー・CBRN・操作・制御喪失の4カテゴリで管理
Governance Source: OpenAI / StartupHub.ai / Techerati — 2026.05.29

OpenAI は本日（5/29）、「Frontier Governance Framework」を公開した。EU AI Act の汎用 AI 規範（Code of Practice for General Purpose AI）とカリフォルニア州の「フロンティア AI 透明性法」に対応するためのガバナンス文書で、既存の Preparedness Framework をベースに公開向けに整理されている。

  • 4つのリスクカテゴリ: ①サイバーオフェンス（攻撃的なサイバー能力）、②CBRN（化学・生物・放射線・核リスク）、③有害な操作（malicious manipulation）、④制御喪失（loss of control）。各カテゴリにリスクティアを設け、段階的な評価プロセスを規定
  • 外部評価の義務化: 新しいリスクティアに接近するモデルに対して独立した第三者評価者によるストレステストを義務付け。内部の「安全諮問グループ（Safety Advisory Group）」に独立意見を提供する
  • 企業評価: Techerati は「OpenAI が自発的なガバナンス文書を公開したことはエンタープライズ AI ガバナンスのシフトを意味する」と評した。一方で「Anthropic の Constitutional AI アプローチや Google の Gemini 安全性ガイドラインとどう違うのか」という比較分析を求める声もある
